# gtme-field-guide

A weekly editorial magazine for GTM engineering signal. Each issue is a
self-contained, horizontal-swipe HTML artifact; the landing page is a
magazine rack listing every issue in reverse chronological order.

This repo is the static shell. Issues are generated by a Claude Code
routine on a weekly schedule (Fridays 08:00 CT), committed directly to
`/issues/`, and auto-indexed by a GitHub Action that rebuilds
`/data/issues.json` from the filesystem.

## Tree

```
gtme-field-guide/
  index.html                  magazine rack — reads /data/issues.json
  vercel.json                 static config (cleanUrls)
  data/
    issues.json               manifest: [{id, date, title, path, preview}]
  issues/                     generated issue-NNN.html files live here
  assets/
    fonts.css                 Playfair · Source Serif · IBM Plex Mono
    shell.css                 index-page styling
  .github/
    workflows/
      rebuild-manifest.yml    deterministic manifest rebuilder
  docs/
    routine-prompt.md         source-of-truth for the weekly routine prompt
```

## Local preview

```sh
python3 -m http.server 8000
# open http://localhost:8000
```

No rewrites, no magic — assets and issues live at literal root paths
(`/assets/...`, `/issues/...`) so URLs work locally and on Vercel
identically.
