Patrick Spychalski: Three Laws of Claude Code Enrichment at Scale

@patrickspychalski

Patrick Spychalski (The Kiln / 2X) shared three hard-won lessons from running Claude Code enrichment at production scale: (1) connect real data APIs — BlitzAPI at $500/mo for LinkedIn enrichment — because Claude cannot hallucinate data into existence, (2) write outputs to Supabase instead of keeping everything in context to prevent bloat and hallucination drift at scale, and (3) be hyper-specific on output format or expect garbage results. He notes that 90%+ of enrichment value lives in the data pipes, not the prompt.

"90%+ of value is in the data pipes"
LINKEDIN
WHY IT MATTERS

The Supabase-as-output pattern is immediately replicable: Claude reads your database, writes structured enrichment results back, and context stays lean. BlitzAPI at $500/mo flat is an underrated budget unlock for agencies doing high-volume LinkedIn enrichment without per-record Clay credits. Build the spec template before you touch the prompt.
